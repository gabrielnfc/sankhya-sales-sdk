**Issue 2 — Fix string injection in GatewayResource.loadRecord (src/resources/gateway.ts line 68):**

Currently: `this.${key} = ${val}` — if val contains quotes or SQL-like chars, it breaks or injects.

Replace line 68:
```typescript
// OLD:
const expression = pkEntries.map(([key, val]) => `this.${key} = ${val}`).join(' AND ');

// NEW — wrap string values in escaped single quotes:
const expression = pkEntries
  .map(([key, val]) => {
    const escaped = String(val).replace(/'/g, "''");
    return `this.${key} = '${escaped}'`;
  })
  .join(' AND ');
```

Note: Sankhya Gateway criteria always use single-quoted string values even for numeric PKs. The `''` escape is standard SQL quoting for embedded single quotes.
