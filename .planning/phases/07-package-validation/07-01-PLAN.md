  <name>Task 2: .npmignore and tsconfig.json updates</name>
  <files>.npmignore, tsconfig.json</files>
  <read_first>.npmignore, tsconfig.json</read_first>
  <action>
    1. Add `README.en.md` to `.npmignore` (per D-02). npm auto-includes anything matching `README*`, so README.en.md leaks into the tarball without this exclusion. Add it on a new line after the existing entries. The current .npmignore content is:
       ```
       src/
       tests/
       docs/
       .env*
       .gitignore
       biome.json
       tsconfig.json
       tsup.config.ts
       vitest.config.ts
       coverage/
       ```
       Add these lines at the end:
       ```
       README.en.md
       .planning/
       examples/
       typedoc.json
       ```
       This ensures examples/, .planning/, typedoc.json, and README.en.md are all excluded from the tarball (per D-02).

    2. Add `"noFallthroughCasesInSwitch": true` to `tsconfig.json` compilerOptions. This flag has zero existing errors (verified in research) and adds free safety. Place it after `"noUnusedParameters": true`. The compilerOptions section currently ends with:
       ```json
       "noUncheckedIndexedAccess": true,
       "noUnusedLocals": true,
       "noUnusedParameters": true
       ```
       It should become:
       ```json
       "noUncheckedIndexedAccess": true,
       "noUnusedLocals": true,
       "noUnusedParameters": true,
       "noFallthroughCasesInSwitch": true
       ```
  </action>
  <verify>
    <automated>node -e "const fs=require('fs'); const ni=fs.readFileSync('.npmignore','utf8'); const tc=JSON.parse(fs.readFileSync('tsconfig.json','utf8').replace(/\/\/.*/g,'')); const ok=ni.includes('README.en.md') && ni.includes('.planning/') && ni.includes('examples/') && tc.compilerOptions.noFallthroughCasesInSwitch===true; process.exit(ok?0:1)"</automated>
  </verify>
  <acceptance_criteria>

    - .npmignore contains `typedoc.json` on its own line
    - tsconfig.json compilerOptions contains `"noFallthroughCasesInSwitch": true`
    - `npx tsc --noEmit` still passes with zero errors
  </acceptance_criteria>
  <done>.npmignore excludes README.en.md, .planning/, examples/, typedoc.json; tsconfig.json has noFallthroughCasesInSwitch enabled</done>
