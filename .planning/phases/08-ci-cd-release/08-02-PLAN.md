Create `.github/workflows/release.yml` per D-01, D-04, D-05, D-06, D-07:

```yaml
name: Release

on:
  push:
    tags: ['v*']

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      id-token: write
    steps:
      - uses: actions/checkout@v6

      - uses: actions/setup-node@v6
        with:
          node-version: 24
          registry-url: https://registry.npmjs.org/

      - run: npm ci

      - name: Quality checks and build
        run: npm run prepublishOnly

      - name: Publish to npm with provenance
        run: npm publish --provenance --access public

      - name: Extract changelog for release
        id: changelog
        run: |
          VERSION=${GITHUB_REF#refs/tags/v}
          NOTES=$(awk "/^## \[${VERSION}\]/{flag=1; next} /^## \[/{flag=0} flag" CHANGELOG.md)
          echo "notes<<EOF" >> $GITHUB_OUTPUT
          echo "$NOTES" >> $GITHUB_OUTPUT
          echo "EOF" >> $GITHUB_OUTPUT

      - uses: softprops/action-gh-release@v2
        with:
          body: ${{ steps.changelog.outputs.notes }}
          generate_release_notes: false
```

CRITICAL details:
- `permissions.id-token: write` is REQUIRED for OIDC Trusted Publishing per D-06
- `permissions.contents: write` is REQUIRED for GitHub Release creation per D-07
- Node 24 is used because it ships with npm 11 natively (required for OIDC, per research Pitfall 3)
- `registry-url: https://registry.npmjs.org/` is required for setup-node to configure .npmrc
- Do NOT set `NODE_AUTH_TOKEN` env var -- OIDC handles auth automatically (per research Pitfall 2)
- `npm run prepublishOnly` runs full pipeline: lint + typecheck + test + build + publint (per research Pitfall 6)
- `npm publish --provenance --access public` uses OIDC (no NPM_TOKEN needed -- classic tokens revoked Dec 2025, per research)
- Changelog extraction uses awk to get content between `## [X.Y.Z]` headings
- `softprops/action-gh-release@v2` creates GitHub Release with extracted notes per D-07
- Version bump is manual per D-05 -- developer updates package.json, commits, creates tag
