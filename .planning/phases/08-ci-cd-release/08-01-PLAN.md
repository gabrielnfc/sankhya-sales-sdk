3. Create `.github/workflows/ci.yml` with the following structure (per D-01, D-02, D-10, D-11, D-12):

```yaml
name: CI

on:
  push:
    branches: ['**']
  pull_request:
    branches: ['**']

jobs:
  quality:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [20, 22]
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-node@v6
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
      - run: npm ci
      - run: npm run lint
      - run: npm run typecheck
      - run: npm run test
      - run: npm run build

  coverage:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-node@v6
        with:
          node-version: 22
          cache: 'npm'
      - run: npm ci
      - name: Run tests with coverage
        run: npx vitest run --coverage --reporter=default --reporter=json-summary --outputFile.json-summary=coverage/coverage-summary.json
      - name: Coverage summary
        if: always()
        run: |
          echo "## Coverage Report" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY
          node -e "
            const fs = require('fs');
            const c = JSON.parse(fs.readFileSync('./coverage/coverage-summary.json', 'utf8'));
            const t = c.total;
            console.log('| Category | Coverage |');
            console.log('|----------|----------|');
            for (const [k, v] of Object.entries(t)) {
              console.log('| ' + k + ' | ' + v.pct + '% |');
            }
          " >> $GITHUB_STEP_SUMMARY
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: coverage-report
          path: coverage/
          retention-days: 30
```

Key details:
- Matrix tests Node 20 + 22 per D-02
- Coverage is a separate job (runs on Node 22 only, not duplicated in matrix) per D-10
- Coverage summary written to `$GITHUB_STEP_SUMMARY` per D-11
- Coverage thresholds enforced by vitest.config.ts per D-12 -- no extra threshold checks needed
- Coverage artifact uploaded per D-10
- npm cache enabled via `actions/setup-node` `cache: 'npm'` (Claude's discretion for caching strategy)

Create `.github/workflows/integration.yml` per D-01, D-03, D-08, D-09:

```yaml
name: Integration Tests

on:
  push:
    branches: [main]

jobs:
  integration:
    runs-on: ubuntu-latest
    timeout-minutes: 15
    env:
      SANKHYA_BASE_URL: ${{ secrets.SANKHYA_BASE_URL }}
      SANKHYA_CLIENT_ID: ${{ secrets.SANKHYA_CLIENT_ID }}
      SANKHYA_CLIENT_SECRET: ${{ secrets.SANKHYA_CLIENT_SECRET }}
      SANKHYA_X_TOKEN: ${{ secrets.SANKHYA_X_TOKEN }}
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-node@v6
        with:
          node-version: 20
          cache: 'npm'
      - run: npm ci
      - run: npm run test:integration
```

Key details:
- Triggers ONLY on push to main per D-03 and D-09 (no cron schedule)
- Node 20 only per D-03
- 4 sandbox secrets as env vars per D-08 (SANKHYA_BASE_URL, SANKHYA_CLIENT_ID, SANKHYA_CLIENT_SECRET, SANKHYA_X_TOKEN)
- NO NPM_TOKEN -- not needed for integration tests (and classic tokens are revoked per research)
- `timeout-minutes: 15` to prevent hanging sandbox tests from consuming runner minutes
- Runs `npm run test:integration` which maps to `vitest run tests/integration/`
