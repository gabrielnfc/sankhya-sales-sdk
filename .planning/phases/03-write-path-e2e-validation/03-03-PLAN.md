Create `tests/integration/e2e-pedido-b2b.test.ts` -- a single sequential integration test that exercises the complete B2B sales flow through the SDK.

Use the established integration test pattern (describe.skipIf, SankhyaClient config from env, beforeAll authenticate). Set test timeout to 120_000 (2 minutes) for the entire describe block since the full flow involves multiple API calls including slow Gateway operations.

The test should be structured as a single `describe` block with sequential `it` tests, each step building on the previous:

**Step 0: Discover sandbox configuration**
In beforeAll (after authenticate):
- `sankhya.cadastros.listarEmpresas()` -- store first `codigoEmpresa`
- `sankhya.cadastros.listarModelosNota()` -- store first `numeroModelo` as `notaModelo`
- `sankhya.cadastros.listarTiposOperacao()` -- store first `codigoTipoOperacao`
- `sankhya.financeiros.listarTiposPagamento()` -- store first `codigoTipoPagamento`
- If any of these return empty, throw Error('Sandbox lacks required config for E2E test') to fail early with a clear message

**Step 1: it('encontra cliente existente')** 
- Call `sankhya.clientes.listar({ page: 0 })`
- Assert data has at least 1 client
- Store first client's codigo as `codigoCliente`
- Log: `console.log('E2E Step 1: Cliente encontrado:', codigoCliente)`

**Step 2: it('consulta produto disponivel')**
- Call `sankhya.produtos.listar({ page: 0 })`
- Assert data has at least 1 product
- Store first product's `codigoProduto`, `volume` (as unidade), and a reasonable `valorUnitario` (use product's preco if available, else 10.00)
- Log the product found

**Step 3: it('verifica estoque do produto')**
- Call `sankhya.estoque.porProduto(codigoProduto)`
- This may return empty array (zero stock not returned by API -- documented in Phase 2)
- Assert the call succeeds (no error thrown)
- If estoque available, log the quantity; if empty, log 'Estoque nao retornado (normal para sandbox)'

**Step 4: it('cria pedido de venda')**
- Build `PedidoVendaInput` using all discovered values:
  - `notaModelo` from Step 0
  - `data`: today in `dd/mm/aaaa` format
  - `hora`: current time in `HH:mm:ss` format
  - `codigoCliente` from Step 1
  - `valorTotal`: quantity * valorUnitario (use quantity 1)
  - `itens`: one item with discovered codigoProduto, quantidade 1, discovered valorUnitario, discovered unidade (or 'UN')
  - `financeiros`: one entry with discovered codigoTipoPagamento, valor matching valorTotal, dataVencimento 30 days out in dd/mm/aaaa, numeroParcela 1
- Call `sankhya.pedidos.criar(pedidoInput)`
- Assert response has `codigoPedido` > 0
- Store `codigoPedido`
- Log: `console.log('E2E Step 4: Pedido criado:', codigoPedido)`

**Step 5: it('confirma pedido')** 
- Call `sankhya.pedidos.confirmar({ codigoPedido })`
- If GatewayError thrown (sandbox lacks fiscal config), catch it, log the error message, and set a flag `confirmaFailed = true`. Do NOT fail the test -- use `console.warn('E2E Step 5: Confirmar falhou (config sandbox):', err.message)` and return
- If success, log confirmation

**Step 6: it('fatura pedido')**
- If `confirmaFailed` is true, skip with: `it('fatura pedido', () => { console.warn('Skipped: confirmar failed'); })` -- or better, check the flag and return early
- Call `sankhya.pedidos.faturar({ codigoPedido, codigoTipoOperacao, dataFaturamento: todayStr })`
- If GatewayError, catch and log (same as Step 5 pattern)
- If success, log faturamento

**Step 7: it('verifica pedido no resultado da consulta')**
- Call `sankhya.pedidos.consultar({ codigoEmpresa })`
- Assert the call returns valid pagination structure
- Note: the created pedido may not be in the first page, so just verify the query works
- Log total pedidos found

**Date helper:** Create a helper function at the top of the file:
```typescript
function formatDate(date: Date): string {
  const dd = String(date.getDate()).padStart(2, '0');
  const mm = String(date.getMonth() + 1).padStart(2, '0');
  const aaaa = date.getFullYear();
  return `${dd}/${mm}/${aaaa}`;
}
```

All tests must use `it.sequential` (or be in a describe with `sequential: true` option) to guarantee execution order. Each test should check that previous steps succeeded via the stored variables -- if a critical variable is missing (codigoPedido not set), skip gracefully.

Important: This test creates REAL data in sandbox. Use identifiable test data (e.g., include "SDK E2E Test" in observacao if the API supports it). The sandbox is disposable so cleanup is not required.
